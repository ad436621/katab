 SECTION 1 — ENCRYPTION ARCHITECTURE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Rules:
- ALL sensitive fields (names, usernames, emails, message content,
  security question answers) MUST be encrypted at rest in the database.
- Encryption algorithm: AES-256-GCM using a server-side secret key
  stored in environment variable ENCRYPTION_KEY (32-byte hex string).
- Decryption MUST happen server-side before sending data to the client.
- The frontend ALWAYS receives and displays plain-text values.
- NEVER store or display raw encrypted strings on any UI page.
- NEVER encrypt on the client side.
- Passwords MUST be hashed using bcrypt (saltRounds = 12) — NOT encrypted.
- Each encrypted field stores: { iv, authTag, ciphertext } as a JSON
  string in a single DB column (TEXT type).

Helper module (create once, import everywhere):
  - encrypt(plainText): returns JSON string
  - decrypt(jsonString): returns plain text
  - All DB reads → decrypt → use; All DB writes → encrypt → store.

 SECTION 5 — SCHEDULED / TIME-LOCKED MESSAGES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Allow any message to be scheduled to "unlock" at a future date/time,
up to years in the future.

5.1 — Database fields (add to messages table):
  - scheduled_unlock_at  DATETIME  (nullable — null = available immediately)
  - is_unlocked          BOOLEAN   DEFAULT false
  - unlock_notified      BOOLEAN   DEFAULT false

5.2 — Composing a scheduled message:
  - In the message compose UI, add a "Schedule unlock" toggle.
  - When enabled, show a date+time picker (datetime-local input).
  - Minimum selectable date: now + 1 minute.
  - Maximum: no limit (years ahead allowed).
  - Display human-readable summary: "This message unlocks on March 15, 2026 at 9:00 AM".
  - Store scheduled_unlock_at encrypted in DB.
  - Message content itself is also encrypted as per Section 1.

5.3 — Unlock logic (server-side scheduler):
  - Run a background job every 60 seconds using setInterval or
    node-cron (cron: "* * * * *").
  - Query all messages where:
      scheduled_unlock_at <= NOW()
      AND is_unlocked = false
  - For each found message:
      a) Set is_unlocked = true.
      b) If recipient has a push subscription and unlock_notified = false:
           Send push notification: "🔓 A scheduled message has unlocked for you!"
           Set unlock_notified = true.

5.4 — Reading/displaying messages:
  - If is_unlocked = false AND scheduled_unlock_at > NOW():
      → Show a locked card UI with a countdown timer and lock icon.
      → Do NOT reveal message content, sender details, or any metadata.
      → Show only: "This message unlocks in X days, Y hours, Z minutes."
      → Animate countdown in real-time using JS setInterval (every second).
  - If is_unlocked = true OR scheduled_unlock_at IS NULL:
      → Show full decrypted message content normally.

5.5 — Countdown UI component:
  - Flip-clock or segmented countdown display (days : hours : mins : secs).
  - Mobile-friendly, centered, visually prominent.
  - Lock icon (🔒) displayed above countdown.
  - Smooth transition animation when message unlocks while user is watching.
