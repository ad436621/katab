## 1. Encryption & Data Handling (Critical Fix)

There is currently a major issue with how encryption is implemented.

Encrypted values (such as usernames or other fields) sometimes appear on the website as encrypted text.
This is incorrect behavior.

### Required Fix

* Encryption must be used **only for database storage and security**.
* The user interface must **always display readable data**.
* Encrypted or hashed values must **never appear directly in the UI**.

### Correct Architecture

* Sensitive data stored in the database → encrypted or hashed.
* Data retrieved from the database → properly decrypted (if encryption) before display.
* Passwords must remain **hashed only** (never decrypted).

### Implementation Rules

* Keep encryption logic in the **backend layer only**.
* The frontend should only receive **clean readable values**.
* Prevent any encrypted strings from rendering in UI components.
