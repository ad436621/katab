── 1A. LOGIN SECURITY QUESTIONS MANAGEMENT ───────────────

The admin must be able to update the security questions used
during the two-step login flow, from the /settings page.

Add a "Login Security Questions" section to /settings with:
  - 3 question text inputs (pre-filled with current values)
  - 3 answer inputs (password type, always empty — never pre-fill)
  - Helper note per answer: "Leave blank to keep current answer"
  - One "Save Security Questions" button

Backend — PUT /api/auth/settings must accept:
  { securityQ1?, securityA1?, securityQ2?, securityA2?,
    securityQ3?, securityA3? }
  - Skip any field that is absent or empty string
  - Normalize answers: trim + lowercase before bcrypt hash (cost 12)
  - Store questions AES-256-GCM encrypted in admin_config table
  - Return { success: true }

Login route POST /api/auth/login must read questions and
answer hashes from admin_config in DB only — not from env vars.
Env vars are used only to seed the DB on first run if empty.

── 1B. BRUTE-FORCE PROTECTION ON LOGIN ───────────────────

Track failed login attempts per IP in a server-side Map:
  - 5 failures within 15 minutes → lock that IP for 15 minutes
  - Return HTTP 429: "Too many failed attempts. Try again in X minutes."
  - Reset counter on successful login
Apply as middleware only on POST /api/auth/login.

── 1C. ERROR HANDLING & LOADING STATES ───────────────────

Every API call in the frontend must handle three states:
  Loading → spinner or skeleton, never a blank screen
  Error   → friendly error card with a "Try again" button
  Empty   → meaningful empty state with a call to action

Add a consistent toast system (use sonner or react-hot-toast):
  - Success toast: green (e.g. "Letter saved", "Settings updated")
  - Error toast:   red   (e.g. "Something went wrong. Try again.")
  - Info toast:    blue  (e.g. "Link copied!", "Draft restored")

All forms must show inline validation errors below each field
(not just a generic top-level toast). Errors appear on field
blur and on submit attempt. Fields with errors get a red border.

── 1D. ENVIRONMENT VALIDATION ON STARTUP ─────────────────

At server startup, before app.listen(), validate:
  ENCRYPTION_KEY must be exactly 64 hex characters
  DATABASE_URL must be set
  VAPID_PUBLIC_KEY, VAPID_PRIVATE_KEY, VAPID_EMAIL must be set

If any are missing or malformed: log a clear descriptive error
and call process.exit(1). Never start with broken config.

── 1E. GLOBAL ERROR HANDLER IN EXPRESS ───────────────────

Add this middleware as the last app.use() in the server:
  app.use((err, req, res, next) => {
    console.error('[ERROR]', err.stack || err.message);
    res.status(err.status || 500).json({
      error: err.message || 'Internal server error'
    });
  });
Wrap every async route handler in try/catch that calls next(err).

── 2A. IN-APP NOTIFICATION BELL ──────────────────────────

Add a new DB table (run migration):
  admin_notifications:
    id         SERIAL PRIMARY KEY
    type       TEXT NOT NULL  -- 'new_reply' | 'letter_read' | 'message_unlocked'
    letterId   INTEGER REFERENCES letters(id) ON DELETE CASCADE
    message    TEXT NOT NULL  -- encrypted, human-readable
    isRead     BOOLEAN DEFAULT false
    createdAt  TIMESTAMP DEFAULT NOW()

Create these notifications automatically on the server when:
  - A recipient submits a reply → type: 'new_reply'
  - A letter is read (unlocked via security questions) → type: 'letter_read'
  - A scheduled message unlocks → type: 'message_unlocked'

Backend routes (admin only):
  GET  /api/notifications          → last 50, ordered by createdAt DESC
  PUT  /api/notifications/read-all → mark all isRead = true
  PUT  /api/notifications/:id/read → mark one as read

Frontend — bell icon in AdminLayout header (top-right):
  - Red badge showing unread count (hidden if zero)
  - Clicking opens a dropdown panel (max-height 400px, scrollable)
  - Each item: icon + decrypted message + relative time ("2 min ago")
  - "Mark all as read" button at top of panel
  - Clicking an item navigates to /letters/:letterId and marks it read
  - Panel closes on outside click
  - Poll GET /api/notifications every 30 seconds while admin is logged in

── 3A. LETTER PREVIEW MODAL IN COMPOSE ───────────────────

Add a "Preview" button next to the submit button in Compose.
Opens a full-screen modal showing the letter exactly as the
recipient will see it: parchment background, royal fonts,
body text, attached image + caption, wax seal decoration.
Security questions are NOT shown (recipient fills those).
"Close Preview" button fixed at top-right. Read-only.
Full-screen on mobile (width: 100vw, height: 100vh, scrollable).

── 3B. AUTO-SAVE DRAFT IN COMPOSE ────────────────────────

Auto-save to localStorage every 30 seconds AND debounced
2 seconds after last keystroke on new letter compose only.

Save: title, body, recipientName, language, questions array,
imageCaption, scheduledUnlockAt state.
Do NOT save imageData (too large for localStorage).

On next visit to /compose (new), show a banner:
  "You have an unsaved draft from [relative time]. Restore it?"
  "Restore Draft" | "Discard" buttons.

Clear draft on: successful submit, "Discard" click,
or navigating to /compose/:id (edit mode uses DB).
Storage key: "royal_letter_draft"

── 3C. SESSION MANAGEMENT IN SETTINGS ────────────────────

Show in /settings:
  - Current session created-at timestamp (formatted)
  - "Log out all sessions" button → deletes all rows from
    admin_sessions table → redirects to /login

If session expires within 30 minutes, show a dismissible
warning banner at top of every admin page:
  "Your session expires soon. Click here to extend."
Clicking calls POST /api/auth/refresh-session which resets
the session expiry to 7 days from now.

── 3E. CLIENT-SIDE IMAGE COMPRESSION ─────────────────────

Before converting uploaded image to base64 for storage,
compress it client-side using the Canvas API:
  - Resize to max 1600px on longest side (preserve aspect ratio)
  - Re-encode as JPEG at quality 0.82 (except PNG — keep as PNG)
  - Show "Compressing..." spinner while processing
  - Show before/after file size (e.g. "4.2 MB → 890 KB")
This reduces DB storage and speeds up letter load times.
